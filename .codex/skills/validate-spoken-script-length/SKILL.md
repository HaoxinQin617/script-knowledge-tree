# 口播稿时长与生成来源校验

把“由指定模型生成”和“约 N 分钟”当作可验证的交付条件，不把它们当作装饰标签。

## 先验证生成来源

1. 区分三件事：当前模型自行撰写、通过工具调用指定模型、把既有文本展示在网站中。
2. 用户指定 ChatGPT、ChatGPT Work 或其他模型时，必须实际调用可用的对应工具、连接器或已获授权的浏览器页面。
3. 保存可核对的生成记录：使用的工具/页面、输入提示词、返回内容或失败状态。
4. 若指定模型不可用，明确说明阻碍并等待用户选择；不得自行撰写后标注“由 ChatGPT 生成”。
5. 不把“网站里写了自动生成”“代码中出现模型名”或“制作了提示词”当作已经完成模型调用的证据。

## 把分钟数换算成篇幅

中文解释型口播默认按每分钟 220 个有效单位估算。一个汉字计 1 个单位，连续英文单词或数字各计 1 个单位；空格和标点不计。

- 3 分钟：目标 660 单位，可接受 560–760。
- 4 分钟：目标 880 单位，可接受 750–1010。
- 5 分钟：目标 1100 单位，可接受 935–1265。

若用户提供自己的语速，以用户语速为准。若稿件包含大量停顿、演示或画面操作，先说明并单独扣除这些时间，不能用停顿掩盖正文不足。

## 生成与验收顺序

1. 从用户资料提取事实、类别和必要术语，禁止为了凑篇幅重复同一句话。
2. 按目标时长写完整口语稿，而不是提纲、摘要或网页卡片文案。
3. 每篇稿件包含开场钩子、解释主线、例子、边界/误区和收束；层级不同则深度不同。
4. 将每篇稿件保存为独立字段或文件，保留标题、层级、标称分钟数和生成来源。
5. 运行 `scripts/check_spoken_length.py`。任何稿件低于最小值都必须扩写后重验；高于最大值则压缩后重验。
6. 在把内容写入网页或宣称完成之前，输出验收表：标题、标称时长、有效单位、预计时长、状态、生成来源。
7. 只有全部状态为 `PASS` 且指定模型调用有真实记录时，才可发布或标记完成。

## 网站任务的额外门槛

- 不要先写短占位文案再给它标上 3/5 分钟。
- 卡片摘要可以短，但详情页的 `script/body/content` 必须存放通过校验的全文。
- 第一层、第二层、第三层分别逐篇校验，不能只验证总字数。
- 筛选、按钮和视觉完成不代表内容完成；内容验收必须早于最终构建和发布。
- 若真实模型调用失败，网站可以保留“等待生成”状态，但不能用伪造完成稿代替。

## 使用校验脚本

脚本接受一个 JSON 文件，内容为对象数组：

```json
[
  {"title":"API","minutes":3,"text":"完整口播稿……","source":"ChatGPT Work tool call id"}
]
```

运行：

```text
python scripts/check_spoken_length.py scripts.json
```

退出码为 0 表示全部通过；退出码为 1 表示至少一篇篇幅不合格；退出码为 2 表示输入结构错误。不得忽略非零退出码。
