# Hermes 同步规则：三层口播知识树

以后处理“上传资料并生成口播知识树网站”的任务时，执行以下固定流程。

1. 每个新 thread 都是独立任务。按收到消息时的日期创建任务，不读取、混合或覆盖其他 thread 的内容。
2. 先完整理解上传资料，提取一个第一层总览、若干第二层类别，以及每个类别真正需要解释的第三层专业名词。不得为了凑数量制造术语。
3. 先生成完整口播正文，再做网页。卡片摘要不能冒充详情页全文。
4. 使用 Humanizer 风格校订：保留事实和篇幅，删除宣传腔、空泛意义、机械三段式、连续短句、假装亲密的开场、套路化总结和其他明显 AI 腔；改成自然、能直接念出的中文口语。
5. 默认按每分钟 220 个有效中文口播单位验收：
   - 3 分钟稿：目标 660，可接受 561–759；
   - 4 分钟稿：目标 880，可接受 748–1012；
   - 5 分钟稿：目标 1100，可接受 935–1265。
6. 第一层、每个第二层类别、每个第三层术语都必须逐篇检查。输出验收表：标题、标称时长、有效单位、预计时长、PASS/SHORT/LONG、生成来源。
7. 任何一篇不是 PASS，都要继续扩写或压缩后重验。全部 PASS 之前不得构建、发布或声称完成。
8. 如果用户指定 ChatGPT Work 或其他模型，必须存在真实工具/页面调用记录。调用失败时保留“等待生成”，不得由其他模型代写后伪称指定模型生成。
9. 网页首屏按日期显示独立任务，并提供“全部、第一层、第二层、第三层”筛选与搜索。进入任务后，正文在左侧或中央，右侧显示下一层独立按钮；第三层保留同级术语切换。必须提供面包屑、返回、复制全文和移动端布局。
10. 视觉采用浅色液态玻璃彩色风格。视觉完成不等于内容完成，发布门槛始终以逐篇内容验收为准。

## 逐稿思维导图与持续优化同步（2026-07-21）

11. 每篇口播必须从完整文字稿提取一张独立的液态玻璃思维导图，不再接受只根据标题生成的通用概念图。导图必须覆盖讲述主线、关键分支、流程或类比、风险/误区和结论锚点。
12. 第一层按“总体逻辑与选择顺序”组织；第二层按“运行步骤、优点、代价、适用条件、区别”组织；第三层按“白话定义、生活类比、作用、误解、记忆句”组织。图片节点的顺序必须能直接作为口播提词路径。
13. 生成前保存 `mindmapOutline` 和完整 `visualPrompt`；生成后检查中文、节点遗漏、连接顺序、16:10 比例与液态玻璃风格。任何错字、伪文字、长段落或无关装饰都要重做。
14. 网页详情页在正文前显示导图并支持点击放大；卡片缩略图与详情页使用同一稳定 `node id` 资源。手机端完整显示导图，不得裁掉分支。
15. 图片必须作为网站本地发布资源保存，禁止引用临时目录或仅本机可见路径；生产发布后逐张检查 HTTP 200 和正确图片类型，不得依赖托管环境缺失的图片优化接口。
16. 以上视觉更新不得改变资料输入、文字生成、口播正文、层级和分钟长度规则。Hermes 在每次同步后继续优化网页，但只能改进导航、层级、响应式、可访问性、图片可读性与加载稳定性，并在构建和线上验证通过后发布。

## 多任务主页同步（2026-07-21）

17. 主页只渲染所有独立任务的第一层根节点，最新上传任务置顶并作为首屏入口；第二层和第三层只能在进入对应父节点后逐层出现。
18. 主页必须提供日期筛选及最新/最早排序。同日任务按完整 `createdAt` 排序，不得覆盖、合并或仅保留一个。
19. 每次新上传完成后，将新任务追加到任务集合而不是替换旧任务，并验证主页只出现第一层卡片、面包屑父子关系正确、第三层不会串到其他类别。

当前经过验证的参考实现：`https://script-knowledge-tree.hxqin.chatgpt.site/`

当前内容验收结果：1 篇 4 分钟总览为 1003 单位；3 篇 5 分钟类别稿为 1198–1253 单位；9 篇 3 分钟术语稿为 566–670 单位；共 13 篇全部 PASS。

## 视觉与多层逻辑同步（2026-07-21）

11. 固定载入 `ui-ux-pro-max`、Figma 设计系统技能、`baoyu-article-illustrator`、`imagegen`、`premium-frontend-ui`、`design-taste-frontend`、`web-design-guidelines`、`frontend-ui-engineering` 与 React 最佳实践。只增强展示层，不得改变资料输入、文字生成模型、口播正文和时长校验模式。
12. 三层导航使用 URL hash 保存当前位置，浏览器返回可恢复上一节点；详情页同时展示面包屑、1-2-3 层进度、当前路径、下一层或同级节点，并保留返回上一层与任务库入口。
13. 每个口播节点必须拥有独立的 16:10 液态玻璃概念图，图片、prompt、alt 与节点 id 一一对应。后续新增节点时，在发布前检查这些资源完整存在。
14. 正文重点识别仅发生在渲染层：专业名词使用轻底纹，关键转折使用玻璃重点段，结论句使用独立结论底纹。复制全文与原始数据仍输出未经标记的原文。
15. Figma 配套以设计 token、可复用卡片、层级徽章、视觉卡与导航节点组件为准；没有目标 Figma fileKey 时不得伪造同步记录，应保存网页端设计系统并等待真实文件后再导入。

## 2026-07-21 · 逐篇线性重制规则

- 后续重制不得统一使用固定“1、2、3 阶段”。Hermes 必须逐篇读取完整口播稿，再决定实际需要的阶段、步骤、判断、分流、风险支线和结论数量。
- 每张图以单向递进逻辑为主，明确表现先后、因果、条件或组成关系；只有原稿确实存在分支或回环时才使用，不得退回中心发散式关键词图。
- 数字只标记真实步骤，数量随稿件变化。定义类、对比类和操作类分别选择适合自身内容的链路，不得为统一视觉虚构相同层数。
