# 口播页面三版本图片展示设计

**状态：** 已由用户确认页面布局，等待规格复核后进入实施计划。

## 目标

每篇口播稳定展示三个互相独立的图片版本，同时保持旧版首图在正文主阅读区的既有位置，并让两个新版图片以不占用正文空间的方式提供预览和放大查看。

## 不可变需求

1. 每篇口播必须恰好关联三个图片版本：旧版首图、新版二、新版三。
2. 旧版首图永久保留，禁止编辑、重新生成、风格迁移或作为新版图片的视觉输入。
3. 新版二和新版三必须只读取该口播的完整文字稿及经审核的结构化提纲，从空白画布独立生成。
4. 禁止读取、引用或编辑网站现有图片来生成新版二或新版三。
5. 三个版本必须使用稳定、显式的数据字段，不得依靠目录命名或渲染顺序隐式推断。
6. 本项目与“卷册”无关。任何内容、图片、示例、测试或界面文字不得使用“卷册”。

## 桌面端布局

- 主阅读栏顶部继续显示旧版首图，位于正文段落之前。
- 右侧栏显示新版二和新版三的预览卡片。
- 两张预览卡必须同时可见，并清楚标注“新版二”和“新版三”。
- 点击预览卡打开对应大图查看；不得替换、覆盖或改变旧版首图。
- 预览卡采用 `button` 或可键盘操作的链接，具备明确的可访问名称和焦点样式。

## 移动端布局

- 旧版首图仍位于正文文字上方。
- 新版二和新版三位于首图下方、正文之前。
- 两张预览卡使用横向滑动区域，保留版本标签和点击放大能力。
- 横向区域不得造成整个页面横向溢出；触控目标不得小于 44 像素。

## 新版图片内容流程

每篇口播先从完整文字稿生成并保存结构化提纲，再生成图片：

1. 一句总标题：只允许概括性短语，不使用正文中的琐碎句子。
2. 若干阶段：阶段数量由内容决定，禁止机械套用固定三阶段。
3. 每个阶段包含必要节点：节点标题必须是可完整朗读的短语。
4. 明确关系：顺序、分支、汇合、对比或循环必须由内容决定。
5. 一句结论：概括整条讲述逻辑，不重复节点列表。

禁止项：省略号标题、截断词语、伪文字、无意义装饰、缺失箭头、箭头指向空白、箭头穿过卡片、文字与图标重叠、仅因版面方便而改变逻辑顺序。

## 视觉规则

- 新版二和新版三延续用户确认的低饱和梦幻玻璃方向，但二者必须是独立设计版本。
- 文字和关系线优先于装饰效果；正文文字使用高对比深色。
- 阶段数字和主要箭头使用灰粉渐变；箭头头部应缩小并与卡片保持安全距离。
- 背景以白色和低饱和粉、紫、蓝为主，减少绿色使用。
- 卡片保留雾面玻璃层次，不使用过度高光、强反射或影响阅读的装饰。
- 所有文字、图标、箭头和容器必须留有明确间距。

## 数据与组件边界

- 建立 `visualVersions` 数据结构，每个节点显式提供 `original`、`version2`、`version3`。
- 旧版首图由主阅读区的 `PrimaryScriptVisual` 渲染。
- 两个新版由独立的 `VisualPreviewRail` 渲染。
- 放大查看由统一的 `VisualLightbox` 处理，保持当前页面上下文并支持关闭、键盘操作和触控。
- 页面不得再次出现把三个版本连续纵向堆叠在正文里的实现。

## 验收标准

### 数据与自动测试

- 每个口播节点恰好具有三个非空、互不相同的图片地址。
- 三个文件都存在且可解析。
- 页面源码中旧版和两个新版由不同组件区域渲染。
- 自动测试验证桌面端右栏含两张新版预览，移动端含横向新版预览区。
- 自动扫描新版图片结构数据，不允许标题包含 `…`、`...` 或截断标记。

### 真实浏览器验收

桌面宽度至少检查 1440 像素：

- 旧版首图位于正文上端。
- 右侧同时显示新版二、新版三。
- 点击两张预览分别打开正确的大图。
- 图片、箭头、文字和卡片没有重叠或裁切。

移动宽度至少检查 390 像素：

- 旧版首图完整显示。
- 两张新版预览可横向滑动。
- 页面本身没有横向滚动条。
- 点击预览与关闭大图均可正常操作。

### 发布门槛

只有同时满足以下条件才能宣布完成：构建通过、自动测试通过、三个版本资源请求均成功、桌面截图验收通过、移动截图验收通过、浏览器控制台无错误。

## 不在本次范围

- 删除或重新生成旧版首图。
- 修改口播稿文字生成模式或用户输入模式。
- 把三个版本改成正文纵向长列表。
- 使用既有图片作为新版生成参考。
