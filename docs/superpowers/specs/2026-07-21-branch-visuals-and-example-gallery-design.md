# 分支讲解图与举例图集设计规范

日期：2026-07-21

## 目标

在现有三层口播知识树中，让每个第一、第二、第三层页面都拥有依据自身完整口播稿重新梳理的讲解图；把含有举例的逻辑小节转换为可翻阅但默认不展开的多格简笔画讲解页；同时调整层级导航和主页“定义”索引，使用户能够直接找到任意层的定义内容。

## 不变约束

- 保留现有口播文字、输入方式和三层数据关系。
- 旧版首图继续放在正文文字上方，不与新版讲解图合并。
- 新版讲解图不得参考或编辑网站已有图片；只允许读取当前节点的完整口播稿和结构化提纲。
- 不强制使用固定三阶段、固定节点数或“逻辑 1、2、3”。实际层数和关系由当前口播内容决定。
- 任何示例不得使用“卷册”，统一改为其他 App、餐厅、旅行、办公或生活场景。

## 一、层级导航位置

### 电脑端

- 从正文主栏移除现有“1 总览、2 类别、3 术语”层级引导。
- 将层级引导放到右侧图片栏顶部，位于新版二、新版三预览之前。
- 当前层高亮；可用的上级和下级节点可点击。
- 层级导航与右侧图片区形成同一块连续阅读区域。

### 移动端

- 层级导航放在旧版首图之后、新版图片横向预览之前。
- 保持单行或可横向滚动，不压缩文字。

## 二、主页“定义”跨层索引

- “全部”和“使用 AI”继续以第一层主题卡为主要展示，避免主页过密。
- 选择“定义”后，显示所有归属定义主题的第一、第二、第三层节点，不再限制层级。
- 每张结果卡显示：标题、摘要、层级、所属第一层主题、日期和预计口播时长。
- 第二、第三层结果可以直接进入对应页面，不要求先经过父页面。
- 日期排序、日期筛选和文字搜索继续有效，并作用于跨层结果。

## 三、每个分支独立讲解图

- 第一、第二、第三层每个口播节点都必须有自己的新版二和新版三。
- 生成输入只包括：节点标题、完整正文、摘要、父子关系和经审核的结构提纲。
- 每张图先提炼主结论，再按因果、流程、对比或递进关系组织节点。
- 标题只放主题名称或高度概括的短语，不把琐碎句子、关系说明或正文截断句当标题。
- 箭头必须完整、方向明确、头部较小，并与文字、图标和玻璃卡保持安全距离。
- 新版二沿用灰粉渐变箭头、梦幻雾面玻璃、线性递进结构。
- 新版三使用白色雾面时间线或另一种独立的线性布局，不能只是新版二换色。
- 两版都突出文字和关系，装饰只能作为轻量背景。

## 四、举例逻辑识别与多格配图

### 识别规则

将以下内容视为需要图解的举例逻辑：

- 明确出现“例如、比如、假设、以……为例”；
- 使用具体人物、产品、餐厅、旅行、办公或生活情境解释抽象概念；
- 通过一组具体输入和结果展示运行方式。

纯定义、总结、抽象对比和没有具体情境的流程不进入举例图集。

### 按逻辑小节分页，不按单句话或单幅画分页

- 先把完整口播中的举例内容按逻辑小节划分，而不是机械地按每个“例如”或每个段落生成图片。
- 一个逻辑小节对应一张讲解页；一张讲解页内部可以包含多幅简笔画。
- 简笔画数量由该小节的运行逻辑决定：需要表示角色、动作、转折、结果或对比时就增加画面，不固定为一幅、三幅或其他数量。
- 同一页通过顺序、分区、箭头或前后对照把多幅画串联起来，单独看这一页就能讲明白对应口播小节的完整逻辑。
- 页面采用“少量关键词文字＋多幅简笔画人物或物体＋必要关系线”的表达，不能出现一整页只有一张孤立插画、无法说明过程的情况。
- 文字只保留场景角色、关键动作、变化和结果，不复制整段口播。
- 简笔画以可识别和关系清晰为先，不追求复杂装饰；整体颜色与网站柔和玻璃体系一致。
- 当一篇口播包含多个不能在同一页清楚表达的逻辑小节时，生成多张讲解页并放入同一个合集。
- 每张讲解页保留独立提示词、所属逻辑小节和覆盖的段落序号，便于以后重生成。

## 五、举例图集交互

- 举例讲解页不在页面上逐张展开。
- 右侧图片栏在新版二、新版三之后显示一个“举例图集 · N 页”合集卡。
- 合集卡只使用一张封面预览，可由第一张多格讲解页与叠层边框构成。
- 点击后进入图集查看器，支持：上一张、下一张、关闭、当前位置“第 X / N 张”。
- 电脑端支持左右方向键和 Escape；移动端支持左右滑动及触摸按钮。
- 切换图片时同步显示该逻辑小节的短标题和对应口播段落范围。
- 没有举例逻辑的页面不显示图集入口。

## 六、数据边界

为每个口播节点维护独立视觉清单：

- `original`：旧版首图；
- `version2`：新版二讲解图；
- `version3`：新版三讲解图；
- `examples[]`：零页或多页举例讲解图，包含图片、短标题、逻辑小节和段落范围。

页面组件只消费清单，不自行猜测文件路径。生成脚本负责根据正文和结构清单创建或更新文件，校验脚本负责检查每个节点的讲解图完整性及举例图数量。

## 七、错误与降级

- 某张举例图生成失败时，保留其他成功图片；合集显示实际成功数量。
- 图片加载失败时显示该图标题和“重新加载”，不让空白区域占据整页。
- 结构图缺失时不回退到其他节点或旧图代替，避免内容错配。
- 举例识别结果和生成清单必须可人工修改，避免自动规则误判后无法纠正。

## 八、验收标准

- 电脑端层级导航位于右侧图片栏顶部，正文主栏不再重复显示。
- 移动端层级导航位于旧首图和新版预览之间。
- 主页选择“定义”后可看到第一、第二、第三层内容，并可直接进入。
- 每个现有节点都有两张从自身口播稿生成的新版讲解图。
- 含举例逻辑的节点按逻辑复杂度生成一页或多页多格简笔画讲解图；每页内部的画面数量由逻辑决定。
- 页面默认只显示一个举例图集入口，不逐张铺开讲解页。
- 图集支持按钮、键盘和触摸翻阅，且不会把全部例图铺开。
- 所有图片路径可在线加载，电脑端和移动端均无横向溢出或遮挡。
- 构建、数据校验、页面测试和线上浏览检查全部通过后才能发布。
